## Описание проекта
Проект реализует модель задачи (Task) с корректной инкапсуляцией и валидацией состояния. Для обеспечения надёжности и защиты инвариантов доменной модели используются пользовательские дескрипторы, вычисляемые свойства и специализированные исключения.

## Функциональность
**Модель задачи (Task):**
- **Атрибуты:**
  - `task_id` (int) — идентификатор задачи (только для чтения)
  - `description` (str) — описание задачи (не может быть пустым)
  - `priority` (int) — приоритет от 1 до 5 (валидация через дескриптор)
  - `status` (str) — статус задачи: 'new', 'in_progress', 'done'
  - `created_at` (datetime) — время создания (только для чтения)
  - `is_ready` (bool) — вычисляемое свойство (готова)

**Валидация через пользовательские дескрипторы:**
- `PriorityDescriptor` — data descriptor для валидации приоритета (тип и диапазон)
- `StatusDescriptor` — data descriptor для валидации статуса (только допустимые значения)
- `DescriptionDescriptor` — data descriptor для валидации описания (не пустая строка)

**Защита инвариантов:**
- Невозможно создать задачу с пустым описанием
- Невозможно установить приоритет вне диапазона 1-5
- Невозможно установить некорректный статус
- ID и время создания доступны только для чтения через `@property`
- Методы `start()` и `complete()` реализуют бизнес-логику переходов между статусами
- При нарушении инвариантов генерируются специализированные исключения

**Демонстрация различий data и non-data дескрипторов:**
- Data дескрипторы (PriorityDescriptor, StatusDescriptor, DescriptionDescriptor) имеют методы `__get__` и `__set__`, что обеспечивает приоритет при присваивании
- Non-data дескрипторы (`@property` для `id`, `created_at`, `is_ready`) имеют только `__get__` и могут быть переопределены через `__dict__`

## Тестирование
Проект покрыт набором тестов, которые находятся в `src2/tests/tests.py`. Тесты проверяют:
- Создание задач с корректными и некорректными данными
- Работу дескрипторов и валидацию
- Жизненный цикл задачи (start, complete)
- Вычисляемые свойства
- Различия между data и non-data дескрипторами
- Специализированные исключения

1. Запуск тестов
```bash
python -m pytest tests/tests.py -v
```

2. Проверка покрытия тестами
```bash
pytest tests/tests.py -v --cov=src2 --cov-report=term-missing
```
