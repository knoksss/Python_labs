## Описание проекта
Проект реализует подсистему приёма задач в платформе обработки задач. Задачи могут поступать из различных источников, не связанных
наследованием, но реализовывающие единый поведенческий контракт.

## Функциональность
**Функциональность:**
- **Модель задачи** — dataclass Task с двумя полями: id (целочисленный идентификатор) и payload (произвольные данные)
- **Контракт источников** — единый интерфейс через typing.Protocol с методом get_tasks() -> list[Task]; проверка соблюдения контракта через issubclass() и isinstance()
- **Типы источников задач:**
  - *TextTaskSource* — чтение задач из текстового файла (формат "id;payload")
  - *GeneratorTaskSource* — генерация задач с фиксированным seed для воспроизводимости
  - *ApiTaskSource* — имитация внешнего API с фиксированным набором задач
- **Обработка** — функции create_task() и print_task() принимают любой источник, реализующий протокол TaskSource, и обрабатывают полученные задачи
- **Логирование** — сохранение всех событий в файл shell.log
- **Расширяемость** — duck typing позволяет добавлять новые классы источников без наследования и модификации существующего кода

## Установка и запуск
1. Установка зависимостей
```bash
pip install -r requirements.txt
```
2. Запуск интерактивной оболочки
```bash
python -m src1.main
```
